1. Database Configuration
Because Docker wasn't running, I configured the backend to use SQLite (sqlite:///./landingforge.db) for instant local testing.

When the FastAPI server starts, SQLAlchemy automatically creates a local .db file inside the backend folder and generates all the tables (users, templates, etc.) if they don't exist.
When you are ready for production, you just change that URL back to PostgreSQL in backend/app/core/config.py.
2. How to Run the Project
To run the servers manually, open two separate terminal windows in your project folder:

Terminal 1 (Backend):

powershell
cd backend
.\venv\Scripts\Activate.ps1
python init_db.py
uvicorn app.main:app --reload --port 8000
Terminal 2 (Frontend):

powershell
cd frontend
npm run dev
3. The Login Process
The Seed: When you ran python init_db.py, it securely hashed the password (admin123) using bcrypt and created the admin@landingforge.com user in the database.
The Request: When you log in at http://localhost:3000/admin/login, the frontend sends your email and password to the FastAPI backend.
The Verification: The backend hashes your entered password, compares it to the database, and if it matches, signs a secure JWT (JSON Web Token).
The State: The frontend receives the JWT and saves it into local storage using Zustand (useAuthStore). Every subsequent request to the backend includes this token to prove you are authenticated!
Feel free to open the dashboard at http://localhost:3000/admin and log in! Let me know if you run into any issues.
